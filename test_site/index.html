RIC Test site for different image use-cases

1. Full viewport width image

Image is intended to cover full viewport width and keep its aspect ratio. Image size changes when viewport is resized.

2. Full container width image

Image is intended to cover full widht of the container and keep its aspect ratio. Image size changes when container width changes. This usually occures when breakpoint changes.

3. Full column width images

These work in the same way as 'Full container width image'

4. Fixed size images

Product image

These images usually are 'thumbnails' and size stays the same in all devices.

This image has fixed size, but it has larger container.

5. Image with fixed width, but overriding max-width to make it fluid

Image is never overflowing the container, because max-width: 100% keeps it inside it. But it has fixed width if container has enough space.

6. Image is smaller than container, but it should fill it

This is used to make sure that layout looks consistent, event if the editor (in CMS) is using low resolution image.
