RIC Test site for different image use-cases

1. Full viewport width image

Image is intended to cover full viewport width and keep its aspect ratio. Image size changes when viewport is resized. original: 2560px x 540px, baseline "standard" quality 9

2. Full container width image

Image is intended to cover full widht of the container and keep its aspect ratio. Image size changes when container width changes. This usually occures when breakpoint changes. original: 1200px x 400px, baseline "standard" quality 9

3. Full column width images

These work in the same way as 'Full container width image' original: 800px x 200px, baseline "standard" quality 9

4. Fixed size images

These images have fixed size, but have larger container. (Except, product image)

These images usually are 'thumbnails' and size stays the same in all devices.

5. Image with fixed width, but overriding max-width to make it fluid

Image is never overflowing the container, because max-width: 100% keeps it inside it. But it has fixed width if container has enough space. original: 800px x 200px, baseline "standard" quality 9

6. Image is smaller than container, but it should fill it

This is used to make sure that layout looks consistent, event if the editor (in CMS) is using low resolution image.original: 200px x 200px, baseline "standard" quality 9
